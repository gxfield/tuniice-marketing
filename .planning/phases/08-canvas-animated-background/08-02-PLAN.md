<task type="auto">
  <name>Task 1: Add visibility pause and reduced-motion gate to CanvasBackground</name>
  <files>src/components/canvas/CanvasBackground.astro</files>
  <action>
Update `src/components/canvas/CanvasBackground.astro` to add three features. All three listeners are registered at module scope (outside `astro:page-load`), so they register only once and persist across navigations.

**1. Page Visibility API pause (module scope):**
```js
document.addEventListener('visibilitychange', () => {
  if (!canvas || !document.contains(canvas)) return;
  if (document.hidden) {
    cancelAnimationFrame(rafId);
    rafId = null;
  } else {
    startLoop();
  }
});
```
This pauses the RAF loop when the user switches tabs and resumes when they return. The `document.contains(canvas)` guard prevents restarting on non-hero pages.

**2. IntersectionObserver for off-viewport pause:**
Add a module-level `let intersectionObserver = null;` variable.

In the `astro:page-load` handler (after canvas is found and before `startLoop()`), set up the observer:
```js
intersectionObserver = new IntersectionObserver(([entry]) => {
  if (entry.isIntersecting) {
    if (rafId === null) startLoop();
  } else {
    cancelAnimationFrame(rafId);
    rafId = null;
  }
}, { threshold: 0 });
intersectionObserver.observe(canvas);
```
In the `astro:before-swap` handler, add cleanup: `if (intersectionObserver && canvas) { intersectionObserver.unobserve(canvas); intersectionObserver = null; }`

**3. Prefers-reduced-motion gate:**
At the top of the `astro:page-load` handler, after finding the canvas element but before any setup:
```js
const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
if (prefersReducedMotion) {
  canvas.style.display = 'none';
  return;
}
```
When reduced motion is preferred, the canvas is hidden entirely. The hero section's existing dark background (or the CSS fallback gradient added in Task 2) shows instead. No RAF loop starts. No ResizeObserver or IntersectionObserver set up.

**Important:** Make sure `startLoop()` guards against double-start. Add `if (rafId !== null) return;` at the top of `startLoop()` to prevent the IntersectionObserver and visibilitychange from stacking loops if both fire in quick succession.
