# Phase 1.2: Remove Header & Update Styling - Context

**Gathered:** 2026-01-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove the header component from the site and overhaul the design system from dark mode to light-mode neo-brutalism with a warm retro palette. All existing components (Button, Card, Footer) get restyled to match. This is a complete visual refresh — not just header removal.

</domain>

<decisions>
## Implementation Decisions

### Color palette direction
- Switch from dark mode to light-mode neo-brutalism
- Warm retro palette: mustard yellow, burnt orange, olive, cream as base colors
- Mixed section backgrounds — different sections get different bold background colors (color blocking)
- Replace neon green accent with a warm accent color (burnt orange, deep red, or teal — Claude picks best fit)
- Pure black for borders and text — neo-brutalism embraces pitch black
- No gradients anywhere — all colors are flat and distinct

### Shadow style
- Black hard shadows with no blur (e.g., 4px 4px 0 #000)
- Classic neo-brutalist maximum contrast approach
- Apply to cards, buttons, and interactive elements

### Layout after header removal
- Content starts immediately near the top — minimal padding, no wasted space
- No header, no navigation chrome — single-page landing, content is the focus
- Body flexbox layout preserved to keep footer at bottom

### Component restyling
- Restyle ALL existing components: Button, Card, Footer
- Complete visual overhaul to match warm retro neo-brutalism
- Thick black borders (existing 3px neo-brutalist tokens stay or increase)
- Zero border-radius (already in design system — keep it)

### Typography
- Keep existing font choices (Space Grotesk / Space Mono) unless they don't fit the warm retro feel
- Bold, oversized headings are core to neo-brutalism

### Claude's Discretion
- Exact hex color codes for the warm retro palette
- Specific accent color choice (burnt orange, deep red, or teal)
- Border thickness adjustments (keep 3px or increase)
- Which sections get which background colors
- Typography weight and size adjustments
- Hover/interaction state details

</decisions>
