<task type="auto">
  <name>Task 1: Create animations.css with scroll animation utilities</name>
  <files>src/styles/animations.css</files>
  <action>
Create `src/styles/animations.css` with scroll-triggered animation utilities:

1. **Base state for animated elements:**
   - Selector: `[data-animate]` (targets any element with the attribute)
   - Initial state: `opacity: 0; transform: translateY(30px);`
   - Transition: `opacity 0.6s cubic-bezier(0.4, 0, 0.2, 1), transform 0.6s cubic-bezier(0.4, 0, 0.2, 1)`

2. **Visible state:**
   - Selector: `[data-animate].is-visible`
   - Visible state: `opacity: 1; transform: translateY(0);`

3. **Stagger delays for grid children:**
   - `[data-animate]:nth-child(1)` -- transition-delay: 0s
   - `[data-animate]:nth-child(2)` -- transition-delay: 0.1s
   - `[data-animate]:nth-child(3)` -- transition-delay: 0.2s

4. **Reduced motion support:**
   - `@media (prefers-reduced-motion: reduce)` block that sets `[data-animate]` to:
     - `opacity: 1; transform: none; transition: none;`
   - This makes content immediately visible with no animation

Keep this file focused and small. No JavaScript here -- just CSS.
  </action>

<task type="auto">
  <name>Task 2: Add ClientRouter and scroll observer to BaseLayout</name>
  <files>src/layouts/BaseLayout.astro</files>
  <action>
Update BaseLayout.astro with three additions:

1. **Import animations.css:**
   Add `import '../styles/animations.css';` alongside existing token and global CSS imports.

2. **Add ClientRouter for view transitions:**
   Import `{ ClientRouter }` from `"astro:transitions"` in the frontmatter.
   Add `<ClientRouter />` inside the `<head>` element (after the title tag).

3. **Add Intersection Observer script:**
   Add a `<script>` block at the end of the body (before closing `</body>`) that:
   - Uses `document.addEventListener('astro:page-load', ...)` to initialize on first load AND after view transition navigation (this is critical -- see research pitfall #1)
   - Inside the handler: create IntersectionObserver with `threshold: 0.15` and `rootMargin: '0px 0px -10% 0px'`
   - Observer callback: when `entry.isIntersecting`, add class `is-visible` to target, then `observer.unobserve(entry.target)` (trigger once)
   - Query all `[data-animate]` elements and observe them

Important: Use `astro:page-load` event (NOT `DOMContentLoaded`) so the observer reinitializes after view transitions. This is the Astro-recommended pattern from the research.

Do NOT change the overall structure of BaseLayout beyond these additions. Keep the existing head meta tags, font links, body structure, and Footer import intact.
  </action>
