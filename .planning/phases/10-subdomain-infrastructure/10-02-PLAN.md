<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Vercel project for bluepriint subdirectory</name>
  <action>
    The user must create a separate Vercel project pointing at the bluepriint/ subdirectory. Claude cannot do this — it requires Vercel dashboard authentication and project creation UI.
  </action>
  <instructions>
    **Step 1: Create new Vercel project**
    1. Go to Vercel Dashboard -> "Add New" -> "Project"
    2. Import the same `tuniice-marketing` git repository
    3. BEFORE clicking Deploy, click "Edit" next to "Root Directory"
    4. Type `bluepriint` as the Root Directory
    5. Framework Preset should auto-detect as "Astro"
    6. Build Command should auto-detect as `astro build`
    7. Output Directory should auto-detect as `dist`
    8. Click "Deploy"
    9. Wait for build to complete — verify it builds the bluepriint site (amber, not green)

    **Step 2: Set environment variable (if you created a Formspree form)**
    1. In the bluepriint Vercel project -> Settings -> Environment Variables
    2. Add: `PUBLIC_FORMSPREE_ID` = your bluepriint form ID from Formspree
    3. Redeploy after adding the env var

    **Step 3: Add custom domain**
    1. In the bluepriint Vercel project -> Settings -> Domains
    2. Click "Add" and enter: `bluepriint.tuniice.com`
    3. Vercel will display a CNAME record to add at your DNS registrar
    4. **Copy the CNAME value** — you'll need it for Step 4

    **Step 4: Add DNS CNAME record**
    1. Go to your DNS registrar/provider for tuniice.com
    2. Add a CNAME record:
       - Name/Host: `bluepriint`
       - Value/Target: [the CNAME value Vercel displayed in Step 3]
       - TTL: default (or 300)
    3. DNS propagation may take a few minutes to a few hours

    **Step 5: Verify SSL**
    1. After DNS propagates, Vercel auto-provisions an SSL certificate
    2. Visit https://bluepriint.tuniice.com — it should load the amber landing page
  </instructions>
  <resume-signal>Type "deployed" once bluepriint.tuniice.com loads correctly, or describe any issues</resume-signal>
</task>
