  <action>
Add a scale transition object to the frontmatter of `story.astro` and wrap the page content in a div with the transition directive.

**1. Add storyTransition object in frontmatter**, after the existing imports:

```typescript
const storyTransition = {
  forwards: {
    old: { name: 'story-out', duration: '0.4s', easing: 'cubic-bezier(0.4, 0, 1, 1)', fillMode: 'both' },
    new: { name: 'story-in', duration: '0.4s', easing: 'cubic-bezier(0, 0, 0.2, 1)', fillMode: 'both', delay: '0.4s' }
  },
  backwards: {
    old: { name: 'story-out', duration: '0.4s', easing: 'cubic-bezier(0.4, 0, 1, 1)', fillMode: 'both' },
    new: { name: 'story-in', duration: '0.4s', easing: 'cubic-bezier(0, 0, 0.2, 1)', fillMode: 'both', delay: '0.4s' }
  }
};
```

Key points:
- 0.4s + 0.4s delay = 800ms total (longer, contemplative feel per user decision)
- Backwards is identical to forwards for scale (zoom has no directional reversal)
- story-out scales UP slightly (1.04) as it exits, story-in scales UP from slightly smaller (0.96) — "focusing in" metaphor

**2. Wrap page content in a transition div:**

Current template:
```astro
<BaseLayout title="Story">
  <StoryHero />
  <Narrative />
  <MusicEmbeds />
  <Connect />
</BaseLayout>
```

Change to:
```astro
<BaseLayout title="Story">
  <div transition:animate={storyTransition}>
    <StoryHero />
    <Narrative />
    <MusicEmbeds />
    <Connect />
  </div>
</BaseLayout>
```

The wrapper div is necessary because `transition:animate` can only be applied to a single element. Without it, each section component would need its own directive. The wrapper ensures the entire page transitions as one unit.
