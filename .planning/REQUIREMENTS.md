# Requirements: tuniice

**Defined:** 2026-01-28
**Core Value:** The site must establish tuniice as a credible, distinctive brand that makes visitors want to follow what's coming next.

## v1 Requirements

### Design System

- [x] **DSGN-01**: Site has a design token system (color palette, typography scale, spacing)
- [x] **DSGN-02**: Reusable component library (buttons, cards, nav, footer)
- [ ] **DSGN-03**: Page transitions, hover effects, and scroll animations
- [x] **DSGN-04**: Custom fonts selected for neo-brutalist aesthetic
- [x] **DSGN-05**: Responsive layout for mobile and desktop

### Navigation

- [x] **NAV-01**: Minimal navigation with logo and hamburger menu

### Home Page

- [ ] **HOME-01**: Hero section with brand name, tagline, and visual impact
- [ ] **HOME-02**: Product preview teasers linking to Products page
- [ ] **HOME-03**: Social links (Bandcamp, SoundCloud, etc.)

### Products

- [ ] **PROD-01**: Coming soon cards for each upcoming product
- [ ] **PROD-02**: Separate sections for apps vs hardware
- [ ] **PROD-03**: Email signup for launch notifications
- [ ] **PROD-04**: Individual product detail pages (even if sparse)

### Music / Art

- [ ] **MUSC-01**: Bandcamp embedded players for releases

### About

- [ ] **ABUT-01**: Bio / story about Greg and the tuniice mission
- [ ] **ABUT-02**: Photo or avatar image
- [ ] **ABUT-03**: Contact info (email or contact form)
- [ ] **ABUT-04**: Technical background, tools, and skills

### Infrastructure

- [x] **INFR-01**: Astro project scaffolded and configured
- [x] **INFR-02**: Deployed to Vercel

## v2 Requirements

### Music / Art

- **MUSC-02**: SoundCloud embedded players
- **MUSC-03**: Visual release grid with artwork

### Products

- **PROD-05**: Full product detail content (screenshots, descriptions, features)

### Documentation

- **DOCS-01**: Documentation page template for apps and devices
- **DOCS-02**: Individual documentation pages per product

### Content

- **CONT-01**: Blog / news section for announcements

## Out of Scope

| Feature | Reason |
|---------|--------|
| Light mode | Dark mode only, matches brand aesthetic |
| CMS / admin panel | Content managed via code, changes infrequently |
| User accounts / auth | No user-facing features requiring login |
| E-commerce / direct sales | Products sold through App Store and other channels |
| Analytics dashboard | Basic Vercel analytics sufficient |
