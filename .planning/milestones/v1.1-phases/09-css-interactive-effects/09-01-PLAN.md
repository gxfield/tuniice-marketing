  <name>Task 2: Add SVG feTurbulence filter to ProductCard for hover distortion on product images</name>
  <files>src/components/ui/ProductCard.astro, src/styles/effects.css</files>
  <action>
1. Modify `src/components/ui/ProductCard.astro`:
   - Add a hidden inline SVG with a `<filter id="card-warp">` def INSIDE the ProductCard template, BEFORE the Card component. Place it as the first child of a wrapper fragment or just before the Card:
     ```html
     <svg width="0" height="0" style="position: absolute;" aria-hidden="true">
       <defs>
         <filter id="card-warp" x="-5%" y="-5%" width="110%" height="110%">
           <feTurbulence type="fractalNoise" baseFrequency="0.015 0.02" numOctaves="2" seed="5" result="noise" />
           <feDisplacementMap in="SourceGraphic" in2="noise" scale="8" xChannelSelector="R" yChannelSelector="G" />
         </filter>
       </defs>
     </svg>
     ```
   - IMPORTANT: SVG filter IDs are document-global. Since multiple ProductCards render on the same page, only ONE SVG filter def is needed. To handle this cleanly: place the SVG def once at the top of the component. Browsers will use the first matching ID in the document, so multiple identical defs are harmless (they just get ignored after the first).
   - Add hover CSS for `.product-image` in the existing `<style>` block:
     ```css
     .product-image {
       transition: filter 0.2s ease;
     }
     ```
   - Gate the turbulence to pointer:fine AND non-reduced-motion using a combined media query directly in ProductCard's style block:
     ```css
     @media (pointer: fine) and (prefers-reduced-motion: no-preference) {
       .product-image:hover {
         filter: url(#card-warp);
       }
     }
     ```
   - NOTE: The hover trigger is on `.product-image` directly, NOT on `.card:hover .product-image`. This is because the ProductCard uses Astro scoped styles — the `.card` class is in Card.astro's scope, not accessible from ProductCard's scoped CSS. Using `.product-image:hover` is simpler. However, since the Card wraps ProductCard's slot content, the product-image IS hovered when the card is hovered (it's inside the card). The slight difference: the filter triggers when the cursor is specifically over the image area, which is actually desirable — it means hovering the text area below doesn't trigger the warp.

   Actually, a better UX is to trigger the filter when the CARD is hovered (entire card area). To achieve this without cross-component scoping, we can use a parent hover. The `.product-link` wraps the card on the products page, but we don't want page-specific styles. The simplest approach: use `:hover` on the product-image and accept that only hovering the image triggers the warp. This is actually what research recommends — apply to image only, and it only triggers on image hover. This is fine.

2. Append to `src/styles/effects.css` (already created in Task 1):
   - No additional rules needed here — the turbulence hover CSS and gating live in ProductCard.astro's scoped styles, and the reduced-motion override is handled by the combined media query gate (effect simply doesn't activate).
