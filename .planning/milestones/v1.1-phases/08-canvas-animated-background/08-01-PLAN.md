  <name>Task 1: Create CanvasBackground.astro with particle system and lifecycle</name>
  <files>src/components/canvas/CanvasBackground.astro</files>
  <action>
Create `src/components/canvas/CanvasBackground.astro` with these parts:

**Template:** A single `<canvas id="hero-canvas" aria-hidden="true">` element with inline styles: `position: absolute; inset: 0; width: 100%; height: 100%; pointer-events: none; z-index: 0;`.

**Script (module-level, inside `<script>` tag):** All logic lives in one script block. Module-level variables persist across navigations.

Module-level variables:
- `let rafId = null;`
- `let canvas = null;`
- `let ctx = null;`
- `let particles = [];`
- `let resizeObserver = null;`

Lifecycle event listeners (registered once at module scope):

1. `document.addEventListener('astro:before-swap', ...)` — Cancel RAF via `cancelAnimationFrame(rafId)`, set `rafId = null`. If `resizeObserver` and `canvas` exist, call `resizeObserver.unobserve(canvas)`.

2. `document.addEventListener('astro:page-load', ...)` — Re-query canvas: `canvas = document.querySelector('#hero-canvas')`. If `!canvas`, return early (not on a hero page). Get context: `ctx = canvas.getContext('2d')`. Set up ResizeObserver. Call `resizeCanvas()`, `initParticles()`, `startLoop()`.

`resizeCanvas()` function:
- `const dpr = Math.min(window.devicePixelRatio || 1, 2);` (cap DPR at 2x)
- `const rect = canvas.getBoundingClientRect();`
- Set `canvas.width = rect.width * dpr;` and `canvas.height = rect.height * dpr;`
- Call `ctx.scale(dpr, dpr);` — CRITICAL: must re-call after every width/height assignment because it resets the transform matrix.

ResizeObserver setup (inside `astro:page-load` handler, stored in module-level `resizeObserver`):
- On resize callback: `cancelAnimationFrame(rafId); rafId = null; resizeCanvas(); initParticles(); startLoop();`
- Call `resizeObserver.observe(canvas);`

`initParticles()` function:
- Read CSS dimensions: `const w = canvas.getBoundingClientRect().width; const h = canvas.getBoundingClientRect().height;`
- Scale particle count to viewport area: `Math.min(Math.max(Math.floor((w * h) / 8000), 20), 80)`
- Create particles array with properties: `x` (random 0..w), `y` (random 0..h), `vx` ((random-0.5)*0.4), `vy` ((random-0.5)*0.4), `r` (random*1.2+0.4)

`startLoop()` function:
- Cache CSS dimensions once: `const w = canvas.getBoundingClientRect().width; const h = canvas.getBoundingClientRect().height;`
- `const linkRadius = 130;`
- Inner `tick()` function:
  - `ctx.clearRect(0, 0, w, h);`
  - Update each particle: move by vx/vy, wrap edges (toroidal — if x<0 set x=w, if x>w set x=0, same for y)
  - Draw each particle: `ctx.arc(p.x, p.y, p.r, 0, Math.PI * 2)` with `fillStyle = 'rgba(0, 255, 65, 0.6)'` (brand accent green)
  - Draw connection lines: nested loop (i, j where j>i), compute distance, if < linkRadius draw line with `strokeStyle = rgba(0, 255, 65, ${(1 - dist/linkRadius) * 0.25})` and `lineWidth = 0.5`
  - `rafId = requestAnimationFrame(tick);`
- Start with `rafId = requestAnimationFrame(tick);`

No styles needed in this component — all styling is inline on the canvas element.
