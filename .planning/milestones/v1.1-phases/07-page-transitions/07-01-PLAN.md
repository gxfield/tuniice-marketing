<task type="auto">
  <name>Task 2: Add default fade transition, global keyframes, scroll reset, and animation replay to BaseLayout</name>
  <files>src/layouts/BaseLayout.astro</files>
  <action>
Make these changes to BaseLayout.astro:

**1. Define the default fade transition object in frontmatter:**

After the existing `const { title } = Astro.props;` line, add:

```typescript
const fadeTransition = {
  forwards: {
    old: { name: 'page-fade-out', duration: '0.3s', easing: 'cubic-bezier(0.4, 0, 1, 1)', fillMode: 'both' },
    new: { name: 'page-fade-in', duration: '0.3s', easing: 'cubic-bezier(0, 0, 0.2, 1)', fillMode: 'both', delay: '0.3s' }
  },
  backwards: {
    old: { name: 'page-fade-out', duration: '0.3s', easing: 'cubic-bezier(0.4, 0, 1, 1)', fillMode: 'both' },
    new: { name: 'page-fade-in', duration: '0.3s', easing: 'cubic-bezier(0, 0, 0.2, 1)', fillMode: 'both', delay: '0.3s' }
  }
};
```

Key points:
- Sequential animation: `delay: '0.3s'` on `new` means the enter animation waits for the exit to complete
- `fillMode: 'both'` prevents flash between exit and enter
- Backwards is identical to forwards for fade (fade has no direction)

**2. Apply transition:animate to `<main>`:**

Change `<main>` to `<main transition:animate={fadeTransition}>`.

**3. Add scroll reset script:**

Add a NEW `<script>` block (separate from the existing observer script) in `<body>`, after the existing `<script>`:

```html
<script>
  document.addEventListener('astro:after-swap', () =>
    window.scrollTo({ left: 0, top: 0, behavior: 'instant' })
  );
</script>
```

This must use `astro:after-swap` (not `astro:page-load`) — fires after DOM swap but before the new page renders, ensuring scroll is reset before transition animations begin.

**4. Remove observer.unobserve to enable animation replay:**

In the existing `<script>` block, remove the line:
```js
observer.unobserve(entry.target); // Trigger animation once
```

This makes `data-animate` elements re-animate on every page visit. The DOM swap from view transitions already removes `is-visible` classes, and the IntersectionObserver re-initializes on each `astro:page-load`. Removing `unobserve` means elements that scroll out and back into view also re-animate, which aligns with the user's decision for replay on every visit.

**5. Add ALL transition keyframes as global styles:**

Add a `<style is:global>` block (separate from the existing scoped `<style>`) with ALL keyframes needed for the entire phase (fade, slide, and scale). Defining them all here ensures Plan 02 page files can reference them without touching BaseLayout:

```css

  from { opacity: 1; }
  to   { opacity: 0; }
}
@keyframes page-fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}

/* Product detail slide (used by products/[slug].astro) */
@keyframes slide-out-left {
  from { transform: translateX(0); opacity: 1; }
  to   { transform: translateX(-5%); opacity: 0; }
}
@keyframes slide-in-right {
  from { transform: translateX(5%); opacity: 0; }
  to   { transform: translateX(0); opacity: 1; }
}
@keyframes slide-out-right {
  from { transform: translateX(0); opacity: 1; }
  to   { transform: translateX(5%); opacity: 0; }
}
@keyframes slide-in-left {
  from { transform: translateX(-5%); opacity: 0; }
  to   { transform: translateX(0); opacity: 1; }
}

/* Story scale/zoom (used by story.astro) */
@keyframes story-out {
  from { opacity: 1; transform: scale(1); }
  to   { opacity: 0; transform: scale(1.04); }
}

  from { opacity: 0; transform: scale(0.96); }
  to   { opacity: 1; transform: scale(1); }
}
```

Do NOT add `@media (prefers-reduced-motion: reduce)` for these keyframes. Astro's ClientRouter automatically disables all view transitions when the user prefers reduced motion. Adding manual overrides would be redundant.
